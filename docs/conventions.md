# 开发规范与操作规程

本文档汇总项目所有开发规范、操作流程和注意事项，供团队成员参考执行。


---


## 1. 代码开发规范

- **先理解再修改**：不允许 AI 在未理解代码逻辑的情况下直接修改代码
- **注释风格**：仅使用单行注释（`//`），不使用多行注释（`/* */`）——多行注释曾导致编译错误
- **算法选型**：使用算法前先查找是否有成熟的开源实现
- **关键代码变更必须附带注释**，包含以下三项：
  1. 修改原因
  2. 修改内容
  3. 影响范围
- **AI 生成的代码变更**必须标注是哪个模型做的修改
- **修改关键代码前必须备份**原文件
- **ROS2 Topic 对接**需确认三要素：消息格式、数据语义、QoS 策略


---


## 2. 参数调整规范

- **先搞懂原理**：调参之前必须理解该参数的作用和影响
- **小幅调整**：注释掉原值，不要删除，保留对比参考
- **大幅调整**：先备份整个配置文件
- **调完立刻测试**：不要拖延几天才测——会忘记改了什么
- **YAML 标记规范**：使用 `# >>>>>>>>>` 行内注释标记已调整的参数，便于快速定位
- **YAML 排版**：主要参数块之间保留三行空行，提高可读性


---


## 3. 文档与日志规范

- **有信息就记录**：问题、TODO、想法、调试发现，全部写进日志
- **已解决的问题**：在备忘中写明「问题描述 + 解决方案」，方便后续查阅
- **时间戳格式**：日志中使用 ROS 系统时间戳（纳秒级）
- **传感器日志命名规范**：参考 `serial_twistctl_node` 第 55-63 行的实现
- **每日工作日志**：必须在 21:15 前完成
- **保留所有 AI 对话记录**：每一段对话都是一个完整的工作场景


---


## 4. Git & GitHub 规范

- **每天下班前必须推送代码**：硬件设备随时可能故障，本地代码可能丢失
- **Commit Message 要求**：
  - 使用英文
  - 按文件逐条描述变更内容
  - 禁止笼统的批量提交（如 "update files"）
- **分支管理**：
  - 禁止直接推送到 `main` 分支，必须通过 PR
  - `main` 分支需要 @kevinlasnh 审批
- **分支命名约定**：

| 前缀 | 用途 |
|------|------|
| `feature/` | 新功能开发 |
| `fix/` | Bug 修复 |
| `tune/` | 参数调整 |
| `docs/` | 文档更新 |
| `experiment/` | 实验性改动 |

## 5. 车辆外出调试规程

### PS2 手柄操作

| 按键/摇杆 | 功能 |
|-----------|------|
| 左方向键 | 切换速度档位 |
| 左摇杆 | 控制线速度 |
| 右摇杆 | 控制角速度 |
| L2 | 切换到串口控制模式 |
| R2 | 切换到手柄控制模式 |
| Y | 使能电机 |
| X | 失能电机（自由滑行） |

### 安全规程

- **自动驾驶期间，必须有一人随时准备按 X 键**（失能电机）
- 车身红色急停按钮：按下停车，旋转释放
- **B 键禁止使用**——紧急刹车会导致车轮严重反转

### 操作流程

1. 首次行驶前：先进入串口控制模式（L2），再使能电机（Y）
2. 在 RViz2 中设定目标点前：确认 `fix_frame` 已设为 `map`
3. 离开实验室前：确认小车充电电源已关闭 + 电池开关已关闭

## 6. ROS2 基本功

- **Topic 对接三要素**：消息格式 + 数据语义 + QoS 策略
- **开发阶段使用 `--symlink-install`**：修改 Python 脚本或配置文件后无需重新构建
- **克隆第三方包时必须指定 `humble` 分支**：其他版本与 ROS2 Humble 不兼容
- **修改包文件结构后**必须同步更新 `CMakeLists.txt`


---


## 7. 工作空间管理

项目运行时产生的数据存放在 `~/fyp_runtime_data/`，与代码仓库分离：

```
~/fyp_runtime_data/
├── logs/              # 运行日志
│   ├── sensor/        # 传感器日志
│   ├── navigation/    # 导航日志
│   └── ...
├── config/            # 运行时配置
└── maps/              # 地图数据
    ├── 2d/            # 2D 栅格地图
    └── 3d/            # 3D 点云地图
```

**代码仓库内的地图**存放于 `src/bringup/maps/`。

## 8. AI Agent 使用规范

- **保留所有 AI 对话记录**：每一段对话都是完整的工作场景
- **一个开发任务 = 一个 Agent 对话**：不要在同一对话中混杂多个无关任务
- **批量重复性修改**：先为 AI 建立工作流程，人工监督执行
- **Copilot 对话必须手动命名**：默认名称无法快速定位历史对话

## 9. 杂项

- PS2 手柄使用 **AAA 电池**（7号电池）
- 读代码时**先看逻辑流程，必要时再看数学公式**
- 系统重启后**记得开启网络端口**
- **不要随意打开车顶绿色盖板**——内部走线复杂，容易碰松接线
