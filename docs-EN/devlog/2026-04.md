### [GPS Live Alignment + Speed Increase to 1.5 m/s]

In response to alignment issues exposed in morning GPS corridor tests, completed four core changes in the evening and passed indoor high-speed verification.

#### Change 1: Calibration Translation-Only (`ebc26e2`)

- **File**: `gps_global_aligner_node.py`
- **Change**: `_solve_calibration_alignment()` removed full 2D rotation solving (centroid + H matrix + atan2), switched to fixed bootstrap theta with translation-only estimation
- **Reason**: With few point pairs and poor spatial distribution, rotation solving produces ~180° flips (e.g., 176.95deg delta in morning test)
- **Effect**: Avoids waypoint calibration rotation flips, maintains rotation stability

#### Change 2: Startup GPS Offset Absorption (`fe3933e`)

- **File**: `gps_global_aligner_node.py`
- **Change**: `_build_bootstrap_alignment()` added `startup_fix` parameter, prioritizes using first stable GPS at power-on as ENU anchor instead of `start_ref` from route file
- **Reason**: Morning test showed startup GPS offset ~4.11m not absorbed, causing systematic offset in first segment targets
- **Effect**: Aligns to actual GPS position on startup day, eliminates startup offset

#### Change 3: Runner Live Alignment (`e73c2bf`)

- **File**: `gps_route_runner_node.py` (core change)
- **Changes**:
  - Removed per-waypoint frozen alignment mechanism
  - Recomputes map coordinates using latest `_latest_alignment` before each subgoal send
  - Added `alignment_shift_cancel_threshold_m: 0.5` parameter: cancels current goal and recomputes when alignment shift exceeds threshold
  - Deleted waypoint calibration request/wait/guard related code (~200 lines)
- **Reason**: Morning test showed per-waypoint frozen alignment caused second segment live alignment rejection (4.96m > 3.0m guard), runner fell back to old alignment, targets continued left offset
- **Effect**: Navigation targets automatically update with GPS real-time refresh, no need to wait for waypoint boundaries

#### Change 4: Speed Increase to 1.5 m/s (`c0ea847`)

  - `vx_max`: 0.7 → 1.5 m/s

- **Reason**: 0.7 m/s still below walking speed, further increase needed
- **Effect**: Theoretical max speed increased to 1.5 m/s
