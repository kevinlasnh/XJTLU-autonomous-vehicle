### [FAST-LIO2 Fatal Jacobian Bug Discovery and Fix]
- CC re-reviewed odom divergence issue; user provided key clue: when car rotates, FAST-LIO2 point cloud map in RViz also rotates

- **Root cause confirmed**: `lidar_processor.cpp:245` point-to-plane Jacobian `hat()` argument uses `state.t_wi` (world position, ~50m) instead of `state.t_il` (extrinsic offset, ~0.04m)

  - Original code uses `offset_T_L_I` (extrinsic translation), which is correct
  - Impact: rotation Jacobian scales with distance from origin (hundreds of times too large) -> incorrect IESKF rotation correction -> map rotation -> odom divergence
- commit `e4945f4`: `Fix FAST-LIO2 Jacobian extrinsic translation bug to stop odom divergence`
  - One-line fix: `state.t_wi` -> `state.t_il`
