# Navigation2 备忘

## 轨迹和路径的区别
1. 轨迹和路径在机器人导航中是两个完全不同的概念，虽然这两者看起来是很相像的
2. 路径（Path）
   1. 空间中一系列几何点的集合
   2. 包含位置 [x,y] 信息，可能还要包含姿态
   3. 物理意义是**我要走这条路**
   4. 生成者为 planner 一个类似全局规划器的东西
   5. 约束条件为静态障碍物和连通性
3. 轨迹（Trajectory）
   1. 空间点 + 时间/速度信息的集合
   2. 包含位置 [x,y,θ] 和速度 [v,w] 信息
   3. 其物理意义是我要在什么时间以什么速度经过这里
   4. 生成者为局部控制器，比如 DWB 控制器
   5. 约束条件为机器人动力学约束（最大速度、加速度），动态障碍物
4. 路径不在乎你开的多快还有在不在漂移，但是轨迹就是具体的驾驶计划，其不仅包含路线，还包含所有的详细速度信息，其明确规定了机器人在每一个时间点的状态

## Nav2 目标点格式
> ---
> header:
>   stamp: <时间戳>
>   frame_id: "map"   # 通常是 map 坐标系（固定地图坐标系）
> pose:
>   position:
>     x: 2.5          # 目标 x 坐标（单位：米）
>     y: -1.0         # 目标 y 坐标（单位：米）
>     z: 0.0          # 通常为 0（2D 导航）
>   orientation:
>     x: 0.0
>     y: 0.0
>     z: 0.7071       # 四元数表示的偏航角（yaw）
>     w: 0.7071       # 例如朝向正东（90度），相当于 yaw = 1.57 rad
> ---
